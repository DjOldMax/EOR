6.2.4. Методы обнаружения стеганографических вложений

Несмотря на то, существует очень много способов включить сообщение в стеганографический контейнер, существует также не меньше способов обнаружить такое включение. Для обнаружения вложения сначала необходимо исследовать формат файла, используемый для вложений, и определить места, где может находиться стеганографическое вложение. Далее следует извлечь данные из этих мест и проанализировать их свойства. Например LSB-метод обнаруживается по аномальным характеристикам распределения значений диапазона младших битов. При этом следует учитывать специфику исследуемых данных, например повторение цветов в данных палитры графического файла говорит о том, что палитра может содержать посторонние данные, или излишне хорошие статистические свойства некоторой части данных могут говорить о том, что используется вложение с шифрованием и т. д.

Фактически перед выполнением стегоанализа аналитик должен разработать модель стегосистемы и проверить ее адекватность реально имеющимся данным. При построении модели надо учитывать:

1) неоднородность последовательностей отсчетов;

2) зависимость между битами в отсчетах (корреляцию);

3) зависимость между отсчетами;

4) неравновероятность условных распределений в последовательности отсчетов;

5) статистику длин серий (последовательностей из одинаковых бит).

Один из перспективных способов обнаружения стеганографических вложений это создание банка мультимедиа файлов, например изображений в формате jpeg. После этого для обнаружения вложения необходимо для картинки передаваемой по каналам связи найти исходное изображение в базе данных (если оно есть) и выполнить сравнение двух файлов. Такие базы данных уже реализованы и используются на практике.

Существует большое количество программных продуктов, предназначенных для обнаружения стеганографических вложений:

• Stegdetect − показывает достаточно уверенные результаты в том случае, если содержание вложения превышает 10 % объема файла-контейнера;

• FTK Imager;

• Stego Suite − позволяет разом уничтожить данные, спрятанные по алгоритму наименее значащих битов, изменяет младшие разряды каждого байта мультимедиафайла на нулевой бит;

• File Signature Header − позволяет не только определить принадлежность какого-либо файла, но и зачастую идентифицировать программу, его создавшую.
