3.5.2. Восстановление части подключа с помощью линейной модели

3.5.2. Восстановление части подключа с помощью линейной модели  

Для восстановления части подключа с использованием линейной модели алгоритма шифрования необходимо использовать достаточное количество данных вида «известный открытый текст/шифртекст». На первом этапе необходимо составить линейные выражения по количеству используемых текстов, соответствующие линейной модели блочного шифра. В первом случае, в выражения подставляются в виде констант значения соответствующих битов.

Вторым этапом для каждого значения искомой части подключа (красный овал на рис. 3.39) выполняем частичное расшифрование последнего раунда блочного шифра для всех зашифрованных текстов. Значения бит перед последней подстановкой, соответствующие выходным битам линейной модели блочного шифра, подставляем в ранее составленные выражения.

Третьим этапом для каждого значения искомой части подключа подсчитываем количество выражений, которые являются справедливыми, и вычисляем вероятность выполнения линейного выражения для этой части подключа путем деления числа истинных выражений на общее количество выражений. Сравнивая практические вероятности линейного выражения, являющегося линейной моделью блочного шифра, для разных значений ключа с теоретическими, приведенными в конце подраздела 2.5.2, выбираем вероятность, наиболее соответствующую теоретической. Часть подключа, с использованием которого была получена эта вероятность, считается верной, обычно ее практическая вероятность выше, чем у всех остальных.

После нахождения одной части подключа последнего раунда необходимо, с использованием других моделей, найти остальные части и потом, расшифровав последний раунд с использованием такой же методики, находим подключи предыдущего. Когда все подключи будут найдены, алгоритм шифрования будет взломан. Как правило, блочные шифры используют некоторую схему расширения ключа пользователя для получения раундовых ключей. При качественном проектировании такого расширения получение ключа пользователя из раундовых ключей невозможно, поэтому в большинстве случаев линейная атака проводится на раундовые ключи блочного шифра. В реальности, вероятности дифференциалов имеют на несколько порядков меньшие значения, чем в приведенном примере, поэтому, чтобы эти слабые вероятностные отклонения проявились, необходимо использовать огромное количество данных требуемого вида.
